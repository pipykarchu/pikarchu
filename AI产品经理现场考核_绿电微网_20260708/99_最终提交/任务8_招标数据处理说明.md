# 任务8：招标数据自动去重、分类、标签筛选 — 设计思路与业务落地说明

## 一、业务背景

绿电微网公司核心业务涉及直流微网、光储充一体化、数据中心供电等领域，日常需从各招标平台（如南方电网阳光采购、中国招标投标公共服务平台等）获取项目线索。

**痛点**：
- 人工每天翻阅数十条公告，效率低
- 相同项目多个平台重复发布，造成重复跟进
- 不同类型项目优先级混淆，高价值线索被淹没

## 二、设计思路

### 处理流程

```
原始招标数据 → 去重 → 关键词匹配分类 → 标签提取 → 优先级判定 → 排序输出
```

### 核心设计决策

| 决策点 | 方案 | 原因 |
|--------|------|------|
| 去重策略 | project_name + buyer + link 联合去重 | 同一项目可能略有标题差异但链接相同 |
| 分类方法 | 基于规则的关键词匹配 | 业务规则明确、可解释、可维护、无需模型 |
| 优先级逻辑 | 多标签命中 + 高优关键词组合判定 | 越贴合公司核心业务的项目越值得优先跟进 |
| 输出格式 | CSV（UTF-8 with BOM） | 兼容 Excel 直接打开中文不乱码 |

### 分类规则（6 大类）

| 类别 | 匹配关键词 |
|------|-----------|
| 直流微网 | 直流、微网、微电网、柔性直流、直流母线 |
| 充电桩 | 充电桩、充换电、充电站、有序充电 |
| 储能 | 储能、电池、削峰填谷、储能系统 |
| 光伏 | 光伏、分布式光伏、太阳能、屋顶光伏 |
| 数据中心供电 | 数据中心、算力中心、机房、UPS、供电系统 |
| 园区电力改造 | 园区、配电改造、节能改造、能源管理 |

### 优先级规则

| 优先级 | 判定条件 |
|--------|---------|
| 高 | 命中 ≥2 个核心标签，或包含"数据中心/直流微网/光储充/零碳园区" |
| 中 | 命中 1 个核心标签且描述明确 |
| 低 | 未命中核心标签，需人工复核 |

## 三、运行结果

## 四、业务落地路径

### 当前价值（已实现）

- **去重**：避免同一项目重复跟进，减少销售/商务撞车
- **分类**：自动识别项目属于公司哪条业务线
- **标签**：快速定位技术关键词，辅助售前快速定方案方向
- **优先级**：高价值线索置顶，商务人员优先响应

### 后续演进方向

```
Phase 1（当前）    Phase 2            Phase 3              Phase 4
规则分类筛选  →  接入公告 API 自动采集  →  AI 语义分类       →  全自动线索流转
CSV 输出      →  企业微信每日推送      →  CRM 线索池对接    →  自动触发售前方案
人工复核      →  标注 → 规则迭代      →  大模型辅助判断    →  智能体方案生成
```

### 技术演进

| 阶段 | 分类方法 | 适用场景 |
|------|---------|---------|
| V1（当前） | 关键词规则匹配 | 业务明确、数据量 <1000/天 |
| V2 | 规则 + TF-IDF 权重 | 需要区分相似项目时 |
| V3 | 大模型语义分类 | 项目描述模糊、新业务类型出现时 |

## 五、AI 辅助开发说明

本脚本使用 Claude 作为 AI 编程助手完成：

- **AI 做了什么**：代码框架生成、分类规则结构化、CSV 读写逻辑、Windows 编码兼容处理
- **人工做了什么**：确定分类标签体系（来自公司业务关键词）、优先级判定逻辑（来自业务理解）、验证输出结果是否符合业务预期

这体现了 AI 产品经理的工作方式：**用 AI 加速实现，用业务判断力保证质量**。
