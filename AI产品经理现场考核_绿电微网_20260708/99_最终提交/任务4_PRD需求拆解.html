1. 背景与目标

1.1 背景

绿电微网公司核心产品「绿电直流运营管理平台」已实现场站监控、计费管理、有序充电、碳资产统计等基础功能。当前三大痛点：

① 调度依赖人工经验：运维人员凭经验设定充放电策略，无法动态响应光伏出力波动和负荷突变，导致弃电率高、用电成本未达最优。
② 碳资产量化粗放：碳减排数据靠月报手工统计，无法实时追踪、精准归因到场站/设备级别。
③ 绿电消纳缺乏可视化决策：运营方不知道哪个时段弃电最多、哪台设备消纳贡献最大。

1.2 目标用户与核心痛点

用户类型典型角色核心场景主要痛点
园区运营方能源管理员、物业经理管理园区光伏+储能+负荷，追求用电成本最低① 调度靠人工；② 不清楚绿电消纳率；③ 碳减排无法量化给业主看
数据中心运维方运维总监、能效工程师7×24高可靠供电，PUE优化，碳中和合规① 峰值负荷大、市电依赖度高；② 碳排放审计要求设备级；③ UPS/储能协同复杂
充电站运营商站长、区域运营经理多站点充电桩调度，平衡服务率和电费① 高峰排队、低谷空置；② 绿电充电比例无法证明；③ 各站碳减排无法独立核算

1.4 价值分析

  • 用户价值：从"看数据做决策"升级为"AI决策+人工监督"，工作负担降低60%+
  • 产品价值：从"监控工具"升级为"智能运营平台"，产品壁垒和定价空间显著提升
  • 公司价值：碳资产量化能力可支撑CCER/绿证交易增值服务，开拓新营收线

3.2 核心功能详述

3.2.1 AI 智能调度引擎

业务流程：

数据采集(15min) → 负荷预测 → 光伏出力预测 → 调度优化求解 → 生成调度方案 → 人工确认/自动执行 → 指令下发 → 执行监控 → 效果评估

功能清单：

功能点描述输入输出验收标准
负荷预测基于历史负荷+气象+日历的短期预测历史负荷、气象、日历未来24h/4h负荷曲线4h预测MAPE≤10%
光伏出力预测基于气象+历史的光伏发电预测气象数据、历史出力未来24h光伏出力曲线日前预测MAPE≤15%
调度策略优化多目标优化：成本/消纳/碳减排预测曲线、电价、设备约束充放电/投切时序表较人工策略电费降低≥8%
调度执行监控追踪指令执行状态和偏差下发指令、实际执行数据偏差告警、执行率统计偏差>10%自动告警
策略效果回溯对比实际与预期收益调度日志、实际电费收益对比报表支持任意时段查询

交互逻辑：

  • 默认"AI建议+人工确认"模式，用户可设为全自动执行
  • 调度策略以甘特图+时序曲线展示，支持手动微调后重新优化
  • 异常时自动回退安全策略并告警
落地约束：调度指令必须经过电气安全校验（不越限、不过载）；储能SOC不低于10%；调度周期≥15分钟。

3.2.2 碳资产实时量化模块

业务流程：

实时用电数据 + 发电数据 → 碳减排计算（基线法） → 设备级归因分配 → 累积碳资产台账 → 核算报告生成 → 碳交易辅助
功能点描述输入输出验收标准
实时碳减排计算按基线排放因子法实时计算绿电发电量、用电量、排放因子实时碳减排量(tCO2e)数据延迟≤1小时
设备级归因按设备贡献分配总减排各设备发电/节电数据设备维度碳贡献表归因偏差<5%
碳资产累积看板展示累计碳资产和趋势历史碳减排数据趋势图、累计值、预测支持日/周/月/年维度
碳核算报告自动生成符合标准的碳报告累计数据+方法学参数PDF/Excel报告符合CCER方法学格式
碳交易时机提示根据碳价趋势建议卖出时机碳市场价格、持有量买卖建议（仅提示）V1.5上线
落地约束：电网排放因子使用国家最新版；碳核算必须可追溯可审计；碳交易提示不构成交易建议（需免责声明）。

3.2.3 绿电消纳分析模块

业务流程：

光伏/风电发电数据 + 负荷消费数据 → 时序匹配计算 → 消纳率统计 → 弃电时段识别 → 瓶颈诊断 → 优化建议 → 反馈给调度模块
功能点描述输入输出验收标准
消纳率时序看板展示各时段绿电消纳率发电、用电时序数据15min粒度消纳率曲线数据延迟≤30min
弃电热力图按时间×设备展示弃电分布弃电时序数据热力图可视化支持日/周维度切换
消纳瓶颈诊断AI识别消纳率低的根因消纳数据+设备状态瓶颈原因列表+建议识别准确率≥80%
优化建议生成基于瓶颈生成调度优化建议诊断结果+设备能力可执行建议列表可直接联动调度模块
消纳对比报告生成周期对比分析报告多周期消纳数据对比图表+结论支持自定义对比周期
落地约束：消纳率计算口径需与电网公司一致；瓶颈诊断需设备参数和拓扑数据；结果需支持导出。

9. AI辅助需求拆解的Prompt思路与人工修正逻辑

9.1 Prompt 设计思路（三层策略）

第一层：业务背景注入

你是一位能源电力行业的产品经理，熟悉分布式能源、直流微网、碳交易。
目标产品是"绿电直流运营管理平台"，服务对象为园区、数据中心、充电站运营方。
当前平台已有：场站监控、计费管理、有序充电、碳资产统计。
需求：迭代升级AI智能调度、绿电消纳分析、碳资产量化三大模块。

第二层：结构化拆解指令

请按以下结构拆解需求：
1. 三类用户（园区/数据中心/充电站）各自的核心痛点（区分差异）
2. 每个AI模块的：功能清单、业务流程、输入输出、验收标准
3. 三个模块之间的联动关系
4. 落地约束：技术、合规、数据、安全

第三层：质量校验提示

检查输出是否存在：
- 功能描述无验收标准的项
- 假设未标注"待确认"的项
- 用户痛点和功能之间无对应关系的项
- 术语不一致或含义模糊的项

9.2 人工修正逻辑

修正环节AI常见问题人工修正方式

技术约束AI不了解15分钟计量周期等电网规范补充电力行业硬约束

9.3 方法论总结

AI辅助PRD = AI结构化初稿(70%) + 行业know-how修正(20%) + 边界约束补充(10%)

核心原则：AI负责结构和效率，人负责边界和判断。AI不做最终决策，产品经理对每一条需求的合理性、可行性、优先级负责。
