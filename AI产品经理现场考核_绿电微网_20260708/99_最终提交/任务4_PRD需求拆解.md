## 1. 背景与目标

### 1.1 背景

绿电微网公司核心产品「绿电直流运营管理平台」已实现场站监控、计费管理、有序充电、碳资产统计等基础功能。当前痛点：

1. **调度依赖人工经验**：运维人员凭经验设定充放电策略，无法动态响应光伏出力波动和负荷突变，导致弃电率高、用电成本未达最优。
2. **碳资产量化粗放**：碳减排数据靠月报手工统计，无法实时追踪、精准归因到场站/设备级别，影响碳交易时效和收益。
3. **绿电消纳缺乏可视化决策**：运营方不知道哪个时段弃电最多、哪台设备消纳贡献最大，无法针对性优化。

本次迭代升级目标：引入 AI 能力，实现**智能调度自动化、碳资产实时量化、绿电消纳可视分析**三大模块。

### 1.2 目标用户与核心痛点

| 用户类型 | 典型角色 | 核心场景 | 主要痛点 |
| --- | --- | --- | --- |
| 园区运营方 | 能源管理员、物业经理 | 管理园区光伏+储能+负荷，追求用电成本最低、绿电自用率最高 | ① 调度靠人工经验，无法应对光伏出力波动；② 不清楚绿电实际消纳了多少；③ 碳减排数据无法量化给业主看 |
| 数据中心运维方 | 运维总监、能效工程师 | 7×24高可靠供电，PUE优化，碳中和合规压力 | ① 峰值负荷大，调度决策慢导致市电依赖度高；② 碳排放审计要求精细到IT负载级别；③ UPS/储能协同调度复杂 |
| 充电站运营商 | 站长、区域运营经理 | 多站点充电桩调度，平衡服务率和电费成本 | ① 高峰时段排队、低谷时段空置；② 绿电充电比例无法证明（绿色品牌需求）；③ 各站碳减排贡献无法独立核算 |

### 1.3 业务目标与成功指标

| 目标 | 衡量指标 | 当前基线 | 目标值 | 统计周期 |
| --- | --- | --- | --- | --- |
| 降低用电成本 | 调度优化后综合电费下降比例 | 人工调度基线 | ≥8% 降幅 | 季度 |
| 提升绿电消纳率 | 本地绿电消纳占总发电比例 | ~70%（待确认） | ≥85% | 月度 |
| 碳资产实时量化 | 碳减排数据从发生到可查询延迟 | T+30天（月报） | ≤T+1天 | 持续 |
| 减少人工干预 | 需人工干预调度策略的频次 | 日均5-10次 | 日均≤1次 | 周 |

### 1.4 价值分析

- **用户价值**：运营人员从"看数据做决策"升级为"AI决策+人工监督"，工作负担降低60%+
- **产品价值**：从"监控工具"升级为"智能运营平台"，产品壁垒和定价空间显著提升
- **公司价值**：碳资产量化能力可支撑CCER/绿证交易增值服务，开拓新营收线

## 2. 范围定义

### 2.1 一句话产品定义

为绿电直流微网运营方提供 AI 驱动的智能调度、绿电消纳分析和碳资产量化能力，实现从"人工经验运维"到"AI自动优化+实时碳资产追踪"的升级。

### 2.2 MVP 功能范围（MoSCoW）

| 优先级 | 功能模块 | 解决的问题 | 说明 |
| --- | --- | --- | --- |
| **Must** | AI智能调度引擎 | 人工调度慢、弃电高、电费贵 | 基于预测的源-荷-储协调优化 |
| **Must** | 碳资产实时量化 | 碳数据月报滞后、无法设备级归因 | 实时碳减排计算+设备级归因 |
| **Must** | 绿电消纳分析看板 | 不知道弃了多少电、消纳瓶颈在哪 | 时序消纳率可视化+优化建议 |
| Should | 调度策略回溯与对比 | 无法评估AI调度效果 | A/B对比人工vs AI策略收益 |
| Could | 碳交易辅助决策 | 何时卖碳收益最高 | 碳价预测+卖出时机建议 |

| 不做项 | 原因 | 后续计划 |

| 移动端App | 优先Web端，运维场景以PC为主 | V1.5 |
| AI模型自训练平台 | 初期用预训练+微调服务即可 | V3.0 |

## 3. 产品方案

### 3.1 功能结构总览

```
绿电直流运营管理平台 AI 升级
├── AI 智能调度模块
│   ├── 负荷预测（短期/超短期）
│   ├── 光伏出力预测
│   ├── 调度策略优化（成本最低/消纳最高/碳减排最大）
│   ├── 调度指令下发与执行监控
│   └── 策略效果评估与回溯
├── 碳资产实时量化模块
│   ├── 实时碳减排计算引擎
│   ├── 设备级碳贡献归因
│   ├── 碳资产累积看板
│   ├── 碳核算报告自动生成
│   └── 碳交易时机提示（V1.5）
└── 绿电消纳分析模块
    ├── 消纳率时序看板
    ├── 弃电时段/设备热力图
    ├── 消纳瓶颈诊断
    ├── 优化建议生成
    └── 消纳率对比报告（日/周/月）
```

### 3.2 核心功能详述

#### 3.2.1 AI 智能调度引擎

**业务流程：**

```
数据采集(15min) → 负荷预测 → 光伏出力预测 → 调度优化求解
    → 生成调度方案 → 人工确认/自动执行 → 指令下发 → 执行监控 → 效果评估
```

**功能清单：**

| 功能点 | 描述 | 输入 | 输出 | 验收标准 |
| --- | --- | --- | --- | --- |
| 负荷预测 | 基于历史负荷+气象+日历的短期预测 | 历史负荷、气象、日历 | 未来24h/4h负荷曲线 | 4h预测MAPE≤10% |
| 光伏出力预测 | 基于气象+历史的光伏发电预测 | 气象数据、历史出力 | 未来24h光伏出力曲线 | 日前预测MAPE≤15% |
| 调度策略优化 | 多目标优化：成本/消纳/碳减排 | 预测曲线、电价、设备约束 | 充放电/投切时序表 | 较人工策略电费降低≥8% |
| 调度执行监控 | 追踪指令执行状态和偏差 | 下发指令、实际执行数据 | 偏差告警、执行率统计 | 偏差>10%自动告警 |
| 策略效果回溯 | 对比实际与预期收益 | 调度日志、实际电费 | 收益对比报表 | 支持任意时段查询 |

**交互逻辑：**

- 默认"AI建议+人工确认"模式，用户可设为全自动执行
- 调度策略以甘特图+时序曲线展示，支持手动微调后重新优化
- 异常时（预测偏差>阈值、设备故障）自动回退安全策略并告警

**落地约束：**

- 调度指令必须经过电气安全校验（不越限、不过载）
- 储能SOC保护：SOC不低于10%（保底容量）
- 调度周期不低于15分钟（与计量周期对齐）

---

#### 3.2.2 碳资产实时量化模块

**业务流程：**

```
实时用电数据 + 发电数据 → 碳减排计算（基线法）
    → 设备级归因分配 → 累积碳资产台账 → 核算报告生成 → 碳交易辅助
```

**功能清单：**

| 功能点 | 描述 | 输入 | 输出 | 验收标准 |
| --- | --- | --- | --- | --- |
| 实时碳减排计算 | 按基线排放因子法实时计算减排量 | 绿电发电量、用电量、电网排放因子 | 实时碳减排量(tCO2e) | 数据延迟≤1小时 |
| 设备级归因 | 将总减排按设备贡献分配 | 各设备发电/节电数据 | 设备维度碳贡献表 | 归因偏差<5% |
| 碳资产累积看板 | 展示累计碳资产和趋势 | 历史碳减排数据 | 趋势图、累计值、预测 | 支持日/周/月/年维度 |
| 碳核算报告 | 自动生成符合标准的碳报告 | 累计数据+方法学参数 | PDF/Excel报告 | 符合CCER方法学格式 |
| 碳交易时机提示 | 根据碳价趋势建议卖出时机 | 碳市场价格、持有量 | 买卖建议（仅提示） | V1.5上线 |

**交互逻辑：**

- 看板首屏展示：今日碳减排、累计碳资产、碳价参考、本月趋势
- 支持drill-down：从总量→场站→设备，层层下钻
- 碳报告一键生成，支持选择核算周期和方法学版本

**落地约束：**

- 电网排放因子使用国家最新发布版本，需支持手动更新
- 碳核算方法必须可追溯、可审计
- 碳交易提示仅为参考，不构成交易建议（免责声明）

---

#### 3.2.3 绿电消纳分析模块

**业务流程：**

```
光伏/风电发电数据 + 负荷消费数据 → 时序匹配计算 → 消纳率统计
    → 弃电时段识别 → 瓶颈诊断 → 优化建议 → 反馈给调度模块
```

**功能清单：**

| 功能点 | 描述 | 输入 | 输出 | 验收标准 |
| --- | --- | --- | --- | --- |
| 消纳率时序看板 | 展示各时段绿电消纳率 | 发电、用电时序数据 | 15min粒度消纳率曲线 | 数据延迟≤30min |
| 弃电热力图 | 按时间×设备展示弃电分布 | 弃电时序数据 | 热力图可视化 | 支持日/周维度切换 |
| 消纳瓶颈诊断 | AI识别消纳率低的根因 | 消纳数据+设备状态 | 瓶颈原因列表+建议 | 识别准确率≥80% |
| 优化建议生成 | 基于瓶颈生成调度优化建议 | 诊断结果+设备能力 | 可执行建议列表 | 建议可直接联动调度模块 |
| 消纳对比报告 | 生成周期对比分析报告 | 多周期消纳数据 | 对比图表+结论 | 支持自定义对比周期 |

**交互逻辑：**

- 看板默认展示今日实时消纳率 + 昨日/上周同期对比
- 弃电热力图支持点击某个色块，展示该时段详情和原因
- 优化建议可一键推送到调度模块执行

**落地约束：**

- 消纳率计算口径需与电网公司统计口径一致
- 瓶颈诊断需要设备额定参数、拓扑关系等基础数据
- 分析结果需支持导出（用于向电网公司/政府汇报）

## 9. 附录：AI辅助需求拆解的Prompt思路与人工修正逻辑

### 9.1 Prompt 设计思路

本PRD的需求拆解过程使用AI辅助，Prompt策略分三层：

**第一层：业务背景注入**

```
你是一位能源电力行业的产品经理，熟悉分布式能源、直流微网、碳交易。
目标产品是"绿电直流运营管理平台"，服务对象为园区、数据中心、充电站运营方。
当前平台已有：场站监控、计费管理、有序充电、碳资产统计。
需求：迭代升级AI智能调度、绿电消纳分析、碳资产量化三大模块。
```

**第二层：结构化拆解指令**

```
请按以下结构拆解需求：
1. 三类用户（园区/数据中心/充电站）各自的核心痛点（区分差异）
2. 每个AI模块的：功能清单、业务流程、输入输出、验收标准
3. 三个模块之间的联动关系
4. 落地约束：技术、合规、数据、安全
```

**第三层：质量校验提示**

```
检查输出是否存在：
- 功能描述无验收标准的项
- 假设未标注"待确认"的项
- 用户痛点和功能之间无对应关系的项
- 术语不一致或含义模糊的项
```

### 9.2 人工修正逻辑

AI生成初稿后，产品经理需进行以下修正：

| 修正环节 | AI常见问题 | 人工修正方式 |
| --- | --- | --- |
| 行业术语 | AI可能混淆"消纳"与"并网"概念 | 用行业标准定义校准 |
| 技术约束 | AI不了解15分钟计量周期等电网规范 | 补充电力行业硬约束 |
| 合规边界 | AI可能建议"自动碳交易"但忽略金融合规 | 标记"本版不做"并说明原因 |
| 数据可得性 | AI假设所有数据都可拿到 | 标注哪些数据需要确认可得性 |
| 优先级判断 | AI倾向堆砌功能 | 用MoSCoW砍到MVP最小集 |
| 用户差异 | AI容易将三类用户混为一体 | 强制拆分用户表，标注差异化需求 |

### 9.3 方法论总结

```
AI辅助PRD = AI结构化初稿(70%) + 行业know-how修正(20%) + 边界约束补充(10%)
```

核心原则：**AI负责结构和效率，人负责边界和判断**。AI不做最终决策，产品经理对每一条需求的合理性、可行性、优先级负责。
