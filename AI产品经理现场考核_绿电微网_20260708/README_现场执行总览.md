用一条清晰主线串起 4 个任务：

> 绿电微网公司需要把招投标线索、客户需求、技术方案、项目运行数据沉淀为可复用的数据资产，并通过 AI 产品、智能体和工作流提升获客、售前、交付、运维效率。

## Claude + Codex 分工操作表

### 分工原则

| 维度 | Claude Code | Codex |
| --- | --- | --- |
| 擅长 | 跨文件上下文理解、方案设计、迭代对话、长链路调试、文档撰写、shell 操作 | 明确的单文件实现、批量机械改动、独立代码片段、并行跑不需上下文的子任务 |
| 不擅长 | 大量重复性代码生成（效率不如并行 Codex） | 需要项目全局上下文的设计决策、多轮需求澄清 |
| 禁忌 | 两个工具同时改同一文件 → 冲突地狱 | 让 Codex 做设计决策（它不了解你的项目约定） |

### 操作节奏

```
1. Claude 规划拆解 → 产出子任务清单
2. 不需上下文的子任务 → 丢 Codex 并行跑
3. 集成 + 验证 + 冲突解决 → 回 Claude
```

### 按任务分工明细

#### 任务4：PRD 需求拆解（35分钟）

| 步骤 | 工具 | 具体动作 |
| --- | --- | --- |
| 1. 需求框架搭建 | **Claude** | 读取输出结构 + 公司关键词，对话确认 PRD 大纲和模块划分 |
| 2. 各模块需求填充 | **Claude** | 逐模块深挖用户故事、功能点、验收标准（需要上下文串联） |
| 3. 表格/字段批量生成 | **Codex** | 给定模板后批量生成：用户痛点表、功能清单表、接口字段表 |
| 4. 业务流程图文字版 | **Codex** | 给定节点列表，生成 Mermaid 流程图代码 |
| 5. 整合校验 | **Claude** | 合并 Codex 产出，校验一致性，补充落地约束 |

#### 任务8：招标数据处理（40分钟）

| 步骤 | 工具 | 具体动作 |
| --- | --- | --- |
| 1. 数据探查+处理方案 | **Claude** | 读 CSV、分析字段、设计去重/分类/标签逻辑 |
| 2. 核心处理脚本 | **Claude** | 写 Python 脚本（需要看数据上下文做判断） |
| 3. 辅助函数/工具代码 | **Codex** | 比如：正则清洗函数、CSV 导出模板、日期格式化工具函数 |
| 4. 运行+调试 | **Claude** | 执行脚本、看报错、修复（Codex 无法执行） |
| 5. 输出说明文档 | **Claude** | 生成处理说明 md |

#### 任务10：全链路 AI 工作流（25分钟）

| 步骤 | 工具 | 具体动作 |
| --- | --- | --- |
| 1. 工作流架构设计 | **Claude** | 对话确认：获客→分析→交付→运维 各环节 AI 切入点 |
| 2. 各环节详细方案 | **Claude** | 需要跨环节上下文理解，逐环节输出 |
| 3. 工具清单表格 | **Codex** | 给定格式后批量填充：工具名、用途、输入输出、频率 |
| 4. Mermaid 流程图 | **Codex** | 给定节点和连接关系，生成标准 Mermaid 代码 |
| 5. 整合成文 | **Claude** | 合并、润色、确保业务逻辑自洽 |

#### 任务11：AI 产品架构（5-10分钟）

| 步骤 | 工具 | 具体动作 |
| --- | --- | --- |
| 全程 | **Claude** | 时间短、需要全局视角，不值得拆给 Codex |

## 现场表达口径

我不是单纯用 AI 生成材料，而是把 AI 当成产品经理的工作台：先用 AI 做初稿、分类和结构化，再人工校验行业逻辑、技术边界、业务落地和可交付成果。
