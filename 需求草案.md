## 零、IP 内核与统一叙事 ⭐

### 0.1 核心叙事：「皮玺玉 × AI 貔貅」副本

整站的灵魂设定 —— **皮玺玉养了一只 AI 貔貅，专门替她囤住生活里的工作流。**

| 元素 | 含义 | 落地方式 |
|---|---|---|
| 貔貅"只进不出" | 囤工作流、囤 Skill、囤好工具 | 每个项目卡片 = 貔貅囤的一件宝贝 |
| 貔貅萌、贪吃 | 把"麻烦事"喂给她，她替你嚼碎 | 俏皮温暖调性的来源 |
| 貔貅招财守运 | 与商业服务、付费定制天然契合 | Services 区文案点睛 |
| 皮玺玉 ↔ 貔貅 谐音 | 名字本身就是 IP，无需额外建立认知 | Logo / 域名 / 文案三位一体 |

**这个内核解决的真正问题：** 采购 / 求职 / PRD / 漫剧 / 健康 这堆跨领域项目能拧成一根绳 —— 不是杂货铺，是"皮玺玉和她的 AI 貔貅一起做的一件件小事"。

### 0.2 调性三件套
- **俏皮**：貔貅萌 + 文案口语 + 微互动彩蛋
- **温暖**：紫粉柔光 + 第一人称 + "替我"/"陪我"动词
- **专业周到**：Linear 极简 UI + 状态徽章 + 数据驱动 + 付费定制兜底

### 0.3 吉祥物视觉基调（已确认）
- **实际方向**：3D 毛绒玩具风萌貔貅（偏方向 B，非原计划方向 A）
- **风格关键词**：奶白蓬松毛 / 大眼睛闪光 / 怀抱宝物 / 泡泡玛特感 / 俏皮温暖
- **主形象（Hero）**：`pixiu-hero.png`（基于"貔貅-26"，正面坐姿抱发光球）
- **次要形象（其他场景）**：`pixiu-secondary.png`（基于"貔貅20-1"，侧坐动态）
- **扩展位置规划**：
  - Hero 主形象（占主导，醒目）
  - Loading / 空状态 / 404（次要场景，用 secondary 或后续生成）
  - 分区标题旁小图、Footer 印章（可选）
- **资产路径**：`content/assets/mascot/`
- **背景处理**：等待用户用 remove.bg 抠图为透明 PNG 后再使用
- **风格警示**：3D 毛绒风偏"萌"，UI 必须用极简留白和大量灰度做平衡，避免整站显得"网红"

## 一、项目定位

### 1.1 一句话定位
**Hi，我是皮玺玉。**

> 主 slogan：**养一只 AI 貔貅，替我囤住生活里的工作流。**
> 副标：产品经理 × AI 实践者，把每个痛点都做成可复用的工具。

### 1.2 站点目标（按优先级）
1. **作品集** —— 把已有 / 在做 / 计划中的 AI 工作流、Skill、应用工具集中展示，让访客 30 秒看懂"她在做什么"。
2. **商业入口** —— 组合式变现：
   - 免费区：可复用 Skill、模板下载、思路文章
   - 付费区：定制咨询、工作流定制、课程（待启动）
   - 私域区：微信 / 小红书二维码引流，长期关系
3. **个人品牌沉淀** —— 长期更新的"皮玺玉数字分身"，对外是简历加强版，对内是项目复盘地图。

### 1.3 目标受众
- **同行 / 产品经理 / AI 爱好者** —— 看作品 + 加微信交流
- **潜在客户（采购、HR、健康赛道从业者）** —— 看案例 → 咨询付费服务
- **HR / 雇主** —— 当成升级版简历看
- **未来学员** —— 等待付费课程上线

## 二、信息架构（IA）

### 2.1 首屏（Hero）
- 大字标题：**Hi，我是皮玺玉**
- 副标题（slogan）：**养一只 AI 貔貅，替我囤住生活里的工作流**
- 一句自我介绍：产品经理 × AI 实践者，把每个痛点都做成可复用的工具（≤ 60 字，可微调）
- 主视觉：**白玉质感萌貔貅** 居右 / 大字居左，貔貅环绕紫粉光晕
- 两个 CTA 按钮：
  - 主按钮 → `看看貔貅囤了啥` 滚动到作品区（文案口语化，呼应叙事）
  - 次按钮 → `约我聊聊` 弹出联系方式 modal（微信 + 小红书 + 邮箱）
- 微互动彩蛋：鼠标 hover 貔貅 → 眨眼 / 小球发光（用 CSS transition 实现，不依赖 Lottie）

### 2.2 主分区（按"领域"归类，非按"类型"）

每个分区 = 一个**领域 / 主题**，包含 3 类资产：Skill、工作流、应用工具。卡片用**状态徽章**区分完成度。

| 分区 | 领域 | 已有 / 在做 / 计划 |
|---|---|---|
| 💼 **求职作战** | 求职 / 职业规划 | 已有：求职作战中心（Live）；计划：简历优化器、JD 匹配 |
| 📋 **PRD 工厂** | 产品经理工具链 | 计划：PRD 生成网页 / Skill |
| 🛒 **采购智能** | 供应链 / 采购预警 | 已有：采购预警看板（Live） |
| 🎬 **内容创作** | AIGC 内容生产 | 计划：漫剧生成、视频特效工作流 |
| 🌸 **她健康** | 女性健康管理 | 计划：减肥规划、经期健康管理 |
| 🧰 **通用 AIGC** | 跨领域可复用 | 沉淀通用 Skill / Prompt 库 |

> **设计原则**：分区按"用户能感知的领域"分（采购、求职、健康……），而不是按"AIGC / 工具 / Skill"这种内部分类分 —— 访客是带着问题来的，不是带着技术分类来的。

### 2.3 卡片状态徽章（关键设计）

避免访客点进去发现 404 的最常见翻车点 —— 每张作品卡片必须打状态：

| 徽章 | 含义 | 视觉 |
|---|---|---|
| `Live` | 已上线可点击体验 | 绿色辉光 |
| `WIP` | 进行中，可能有截图但未上线 | 紫色（主色） |
| `Idea` | 想法阶段，欢迎共创 | 灰色低调 |
| `Paid` | 付费服务 | 粉色 + 价格标签 |

### 2.4 商业服务区（Services）
组合式三种入口并列展示：

1. **免费资源**（吸引）：可复用 Skill 包、提示词模板、复盘文章 → 一键复制 / 下载
2. **付费服务**（转化）：定制咨询 / 工作流外包 / 课程预售 → 引导填表或私聊报价
3. **私域引流**（沉淀）：微信 + 小红书 + 公众号二维码 → 长期关注

### 2.5 关于 / About
- 个人简介（来历、为什么做这些）
- 联系方式（邮箱、微信、小红书、GitHub）
- 简历下载（PDF）

### 2.6 页脚
- 版权 + 备案号占位
- 友情链接（如有）
- 最后更新时间（自动从 git 取）

## 四、UI / 视觉规范

沿用全局 `~/.claude/design-system.md` Linear + 紫粉渐变规范，**叠加貔貅 IP 视觉系统**：

### 4.1 基础规范（沿用全局）
- **基调**：90% 灰度 + 紫粉点缀，深浅主题双向支持
- **主题策略（已确认）**：默认跟随系统 `prefers-color-scheme`，用户可手动切换并 localStorage 持久化
  - **浅色主题（默认推荐）**：奶白底色，配合貔貅暖色，整体温暖柔和
  - **深色主题（可切换）**：保留 Linear 风深色，貔貅在深色卡上加微辉光衬托
- **强调色**：紫(#A78BFA) → 粉(#F472B6) 渐变用于 hero 标题、CTA 主按钮、状态徽章 Live
- **动画**：入场用 `fade-in-up`，hover 用 `transition-all duration-300`
- **首屏装饰**：紫粉光晕（blur + 低透明度），不堆砌，留白要够
- **字体**：中文用系统默认（PingFang / 微软雅黑），英文用 Inter

### 4.2 貔貅 IP 视觉系统 ⭐
- **主形象配色**：白玉半透明（避免土味的金色 / 翡翠绿），与紫粉渐变协调
- **出现位置规划**：
  - Hero 区：主形象（坐姿 / 正面 / 抱发光小球）
  - 分区标题左侧：小尺寸貔貅 icon（每个分区一个表情，可选）
  - Loading 状态：旋转的貔貅 + "貔貅正在囤..."
  - 空状态：发呆貔貅 + "这里还没囤上"
  - 404：迷路貔貅 + "貔貅找不到这个宝贝了"
  - Footer：缩小版貔貅做品牌印章
- **资产命名约定**（统一放 `content/assets/mascot/`）：
  - `pixiu-hero.png` - 主形象
  - `pixiu-loading.png` - 加载用
  - `pixiu-empty.png` - 空状态
  - `pixiu-404.png` - 404 页
  - `pixiu-icon.png` - 通用 favicon / 印章小图
- **首期最小集**：只要 1 张 `pixiu-hero.png` 就能上线，其他位置先用占位 emoji（🐾），后续陆续补
- **主标题字色**：纯白大字（不用紫粉渐变字），紫粉只做装饰光晕，更高级

## 五、技术栈

- **前端**：Vue 3 + `<script setup>` + Vite 7+
- **样式**：Tailwind CSS 4.x（沿用全局 design system）
- **内容解析**：`vite-plugin-md` 或 `gray-matter` + `marked` 加载 `content/projects/*.md`
- **路由**：单页滚动 + 锚点（不引入 vue-router，简单优先）；项目详情用 modal 或独立页二选一
- **部署**：Vercel / Netlify / GitHub Pages（待定）
- **版本控制**：遵循全局 cache busting 规范，`?v=x.x.x` 同步

> ❗ 不引入：Pinia / Vuex（无需全局状态）、TypeScript（保持轻量）、SSR（无 SEO 必要时不需要）。

## 六、商业模式落地细节

### 6.1 免费区 → 付费区漏斗

```
免费 Skill 包 → 复制即用 → "想要定制版？" → 私聊 → 付费服务
```

每个免费资源卡片底部带一行小字提示："👉 想要定制版本？[联系皮玺玉]"

### 6.2 付费服务呈现

- 不要在网页直接挂支付按钮（前期）
- 用"咨询"按钮 → 跳转表单（Tally / Feishu / 微信）
- 付费定价可以"区间制"展示（如：定制工作流 ¥xxx 起）

### 6.3 私域入口

- 浮动按钮（右下角持续可见）："+ 加皮玺玉"
- 点击 → 弹 modal：微信二维码 + 小红书二维码 + 邮箱
