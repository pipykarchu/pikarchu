## 2. 面试前优先级

| 优先级 | 任务 | 为什么重要 | 成功标志 |
| --- | --- | --- | --- |
| P0 | 选出 2 个最强项目重点讲 | 面试时间有限，不能平均用力 | 能在 3 分钟内讲清每个项目 |
| P0 | 跑通首页 → 作品 → 客服 → 留言 → 后台链路 | 证明不是静态简历页 | 全链路 10 分钟内完成 |
| P0 | 准备 AI 不可用兜底说法 | 面试现场网络不稳定 | 不因客服失败影响演示 |
| P1 | 补真实二维码和邮箱 | 让联系入口可信 | 联系弹窗可用 |
| P1 | 准备项目截图/录屏 | Demo 失败时可兜底 | 每个 Live 项目有备用素材 |
| P1 | 明确当前版与小程序商业化边界 | 避免被追问商业闭环时失焦 | 能一句话说明“当前演示，后续立项” |
| P2 | 补真实运营数据 | 提升说服力 | 后台有可展示事件和留言 |
