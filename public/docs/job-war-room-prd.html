1. 背景与目标

求职过程不是一次性写简历，而是持续整理信息、判断岗位、定制材料、准备面试、复盘反馈的复杂链路。求职作战中心把这些动作整理成本地优先的 AI Agent 工作台，让求职材料、决策过程和交付产物可追踪、可复用、可演示。

用户类型核心场景主要痛点
转岗/求职中的产品经理看 JD、改简历、准备面试简历和项目讲法无法稳定匹配岗位
AI 工具实践者向面试官展示 AI 产品和工程落地项目很多但难讲成一条业务链路
面试官/HR打开 Demo 和 PRD 看项目结构不想看零散截图和难复现本地服务

2. 范围定义

一句话定义：在本地优先和隐私隔离前提下，为求职者提供个人档案、路径推演、JD 决策、简历定制、面试演练、学习作战和作品集展示的 AI Agent 工作台。
优先级功能解决的问题说明
Must个人档案与素材库避免每次重新介绍自己保存经历、项目、技能、偏好和约束。
MustJD 解析与决策建议避免盲投和 AI 编造解析岗位要求、匹配证据、风险点和行动建议。
Must简历与面试产物生成把判断转成可投递材料输出简历建议、面试手册、STAR 讲法。
Should学习作战与作品集展示把缺口转行动，把项目转证据输出学习任务和公开讲法。
本版不做原因后续计划
自动投递岗位涉及平台账号、反爬、隐私和误投风险仅保留人工确认后的外部跳转方向。
真实公司信息自动断言容易生成过期或错误事实需要来源链接和人工确认。
多用户 SaaS 权限体系MVP 是本地优先个人工作台后续评估账号、同步和团队版。

3. 产品方案

flowchart TD
  A["个人档案"] --> B["路径推演"]
  A --> C["JD 解析"]
  C --> D["决策建议"]
  D --> E["简历定制"]
  D --> F["面试手册"]
  F --> G["面试演练"]
  D --> H["学习作战"]
  A --> I["作品集展示"]
  I --> J["公开 Demo / PRD"]
对象核心字段用途
Profile经历、项目、技能、偏好、限制作为所有生成任务的事实来源。
JD岗位、公司、职责、要求、来源时间用于解析和匹配。
Decision匹配度、证据、风险、行动建议帮用户决定是否投入。
Interview Pack自我介绍、项目讲法、STAR、追问形成面试准备包。

4. 全局规则

  • 真实档案和求职材料优先本地保存，不默认上传公开站点。
  • 公开 Demo 必须使用虚拟档案和只读数据。
  • 所有岗位和公司事实必须说明来源或标记待确认。
  • 简历产物要说明“为什么这样改”，不是只给结果。
  • 模型失败时不返回硬编码假结果，必须提示失败或降级。

6. 需求详情

R1. 个人档案与素材库

档案字段分为已确认事实、可包装表达和待确认材料。AI 不得把待确认材料直接写入最终简历；演示档案与真实档案必须隔离。

R2. JD 解析与决策建议

JD 输出必须区分原文信息、系统推断和待确认。匹配建议必须引用档案里的项目或经历作为证据，不输出录取概率、薪资预测等无来源结论。

R3. 简历定制与面试演练

简历建议必须包含修改前后方向和修改理由；面试手册覆盖自我介绍、项目 STAR、风险问题和反问建议。
