1. 背景与目标

个人站已经有个人介绍、作品展示、联系方式和留言能力，但访客需要自己浏览页面寻找信息。智能客服把这些资料变成对话入口，降低理解成本。

目标衡量方式成功标志
快速理解个人站客服打开量、问题命中率访客 1 分钟内获得个人介绍、作品概览或联系入口
提高转化联系按钮点击、留言提交客服窗口能自然引导到联系弹窗

2. MVP 范围

功能说明优先级
智能客服浮窗页面右下角客服入口，点击展开聊天窗口P0
资料问答基于 profile、projects、services 回答站点范围内问题P0
留言/联系入口客服窗口内打开现有 ContactModalP0
兜底回复无 Key 或模型失败时引导留言P0
访问频控后端对聊天接口做简单限流P1
本版不做人工坐席、用户登录、长期聊天记录、复杂知识库后台和自动报价。

3. 用户流程

  • 访客打开个人站。
  • 点击右下角客服入口。
  • 选择快捷问题或手动输入问题。
  • 前端把问题发送到 /api/chat。
  • 后端读取站点资料并调用模型生成回答。
  • 访客需要继续沟通时点击留言 / 联系。

4. 核心需求

需求验收标准
浮窗交互可展开、可关闭，移动端不遮挡主要内容
资料问答能回答“你是谁”“有哪些作品”“怎么联系”
范围控制资料外问题说明不确定并引导留言
安全API Key 只在后端环境变量中读取

5. 风险与对策

风险影响对策
API Key 泄露产生费用或被滥用只放 .env，不提交 GitHub
AI 编造信息误导访客系统提示限制只基于站点资料回答
模型不可用聊天失败返回本地兜底回复并引导留言
调用频繁成本上升后端限流
