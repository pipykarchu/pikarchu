皮玺玉个人 IP 站 SOP

0. 使用说明

这份 SOP 给自己和协作者照着做，目标是让面试演示版稳定、可维护、可复盘。

| SOP 类型 | 使用时机 | 成功标志 |

| --- | --- | --- |

| 面试前检查 SOP | 面试前 1 天或当天 | 10 分钟演示链路能完整跑通 |

| 内容维护 SOP | 新增作品、改服务、换二维码时 | 不改业务代码也能更新页面 |

| 本地运行 SOP | 需要展示客服、留言、后台时 | 前台和 API 都可用 |

| 上线前检查 SOP | 准备公开访问时 | 无默认密码、无 Key 泄露、基础隐私说明完整 |

1. 面试前检查 SOP

目的

确保面试现场不会因为链接、图片、后台、客服或文档问题打断演示。

步骤

| 步骤 | 操作 | 成功标志 |

| --- | --- | --- |

| 1 | 打开首页 | 首屏文字、吉祥物、按钮正常显示 |

| 2 | 点击“看看貔貅囤了啥” | 页面滚动到作品区 |

| 3 | 打开 2 个代表项目 | 项目详情能展示背景、亮点、技术栈 |

| 4 | 点击 Live 项目 Demo | Demo 能打开；打不开就准备截图 |

| 5 | 打开智能客服 | 客服浮窗能展开 |

| 6 | 问“有哪些作品” | 返回作品相关答案或兜底说明 |

| 7 | 打开联系弹窗 | 表单、二维码、邮箱正常 |

| 8 | 提交测试留言 | 后台能看到留言 |

| 9 | 进入 `/admin` | 能登录并看到留言和事件 |

| 10 | 打开 PRD HTML | 能讲清当前是面试演示版，小程序商业化是后续方向 |

2. 内容维护 SOP

目的

以后改内容优先改数据，不动业务代码，降低维护成本。

| 事项 | 文件 | 操作 | 成功标志 |

| --- | --- | --- | --- |

| 改个人介绍 | `content/profile.json` | 修改 `greeting/slogan/intro/contact` | 首页和联系弹窗同步更新 |

| 加新作品 | `content/projects.json` | 复制一个项目对象并修改字段 | 对应分区出现新卡片 |

| 改作品状态 | `content/projects.json` | 修改 `status` | 徽章变成 Live/WIP/Idea/Paid |

| 改分区 | `content/categories.json` | 修改 `title/desc/order/accent` | 分区名称、顺序或颜色变化 |

| 改服务文案 | `content/services.json` | 修改服务对象 | 服务区文案更新 |

| 换二维码 | `public/qrcodes/` | 替换 `wechat.png` 或 `xhs.png` | 联系弹窗显示新二维码 |

| 换吉祥物 | `public/mascot/` | 替换对应 PNG | 首页或客服入口显示新图 |

3. 本地运行 SOP

目的

如果只看静态页面，可以直接打开；如果要展示客服、留言和后台，必须启动 API。

| 场景 | 操作 | 成功标志 |

| --- | --- | --- |

| 只看 PRD | 打开 `皮玺玉个人IP站_PRD.html` | 文档正常显示 |

| 只看前台静态效果 | 运行 `npm run dev` | 首页可访问 |

| 展示客服/留言/后台 | 运行 `npm run dev:full` | 前台和 API 都可用 |

| 检查 API | 打开 `/api/health` | 返回 `ok: true` |

| 登录后台 | 打开 `/admin` 并登录 | 能看到留言和数据 |

风险

  • 如果没有配置 AI Key，客服会走本地兜底，这不是坏事，可以讲成“防幻觉和无 Key 降级策略”。
  • 如果没有配置邮件，留言仍然保存，只是不会邮件提醒。
  • 不要把 `.env`、真实 API Key、邮箱授权码提交到 Git。

    • 4. 上线前检查 SOP

    目的

    从面试演示版转公开访问时，先把安全、隐私、配置补齐。

    | 检查项 | 操作 | 成功标志 |

    | --- | --- | --- |

    | 后台账号 | 配置 `ADMIN_USER`、`ADMIN_PASS` | 默认账号密码不可用 |

    | Token 密钥 | 配置 `ADMIN_SECRET` | 重启后签名逻辑稳定 |

    | AI Key | 配置后端环境变量 | 前端代码搜不到真实 Key |

    | 邮件提醒 | 配置 Resend 或 SMTP | 新留言能收到邮件 |

    | 隐私说明 | 页面或文档说明数据用途 | 访客知道留言和埋点用途 |

    | 二维码 | 上传真实二维码 | 联系弹窗可扫码 |

    | 移动端 | 用 390px/430px 检查 | 不遮挡主要内容 |

    | 构建 | 运行 `npm run build` | 构建成功 |
