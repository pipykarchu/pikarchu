1. 项目定义

1.1 一句话定义

《娃娃仙》是一部 12 集竖屏真实恐怖志怪漫剧：女儿缠着母亲讲家族旧事，母亲讲起自己小时候被一个粗布娃娃多次显灵保护，最终娃娃仙为斩旱魃耗尽灵力，只留下一颗象征重生的姜。

1.2 核心看点

看点表现方式
家族旧事感现实层从女儿追问开始，母亲以“睡前故事”方式讲述
民俗恐怖娘子关一带旧村、竹林深坑、拦路鬼、旱魃、红毛菌丝、火堆梦境
护主报恩姥爷救道士种善因，娃娃仙多次救“妮儿”
代价递增每救一次，娃娃少一根辫子；最后辫子全无，灵力耗尽
情绪余味道士取回娃娃，留下姜；现实层回到墓前，等待姜发芽

1.3 目标观众

观众类型核心期待适配重点
B 站民俗故事观众有完整故事、有细节、有情绪后劲旁白质感、章节感、评论讨论点
志怪恐怖观众诡异事件、民间禁忌、压迫氛围竹林、拦路鬼、旱魃画面要有记忆点
情绪向观众善良被回报、守护与牺牲、结尾希望娃娃仙的“少一根辫子”要反复强化
短剧平台观众每集开头有钩子，结尾有追更点抖音/红果版每集前 3 秒必须有强冲突

2. 叙事方案

2.1 总体叙事结构

采用“现实层包裹主线回忆”的讲述方式。

现实层：女儿缠着阿妮讲娃娃仙的故事
  -> 回忆主线：姥爷救道士，娃娃仙多次显灵护主
  -> 高潮：娃娃仙斩旱魃，耗尽灵力
  -> 现实层回归：清明墓前，姜没有发芽，但家人仍相信她会回来

双线职责固定如下：

线索负责内容出现节奏作用
现实线2026 年清明前一晚，阿妮给姜苗、姜生讲娃娃仙；第 12 集清明扫墓，看墓前姜是否发芽第 1 集开口，第 12 集收束；中段尽量不频繁切回让“红布姜”“孩子名字带姜”“扫墓看姜”形成闭环
故事线姥爷救道士、五辫娃娃来到小妮儿身边、娃娃仙多次显灵护主、斩旱魃、婚礼被道士取回第 1 集进入后持续推进到第 11 集承担主要恐怖志怪事件和情绪牺牲

2.2 现实层设置

项目设定
讲述者成年后的妮儿，已为人母
听故事的人弟弟姜生主问，女儿姜苗说自己从小听姥姥讲到都会背；两个名字都带“姜”以呼应结尾
现实层时间2026 年清明前一晚，第二天要早起给姥姥姥爷上坟
进入主线方式阿妮给两个孩子盖被子，说“明儿清明，还得早起去给你姥姥姥爷上坟”；姜生缠着要听娃娃仙
现实层道具床头柜里上密码锁的小盒子、红布包着的干姜、旧布娃娃记忆、清明墓前泥土
回归现实方式第 12 集回到清明，女儿问姜会不会发芽，妮儿回答“这世上没有什么是不可能的”

2.3 主角称呼规则

原文使用“妈妈”是讲述视角，不适合剧中主线一直出现。剧中主角暂定：

阶段称呼用法
幼年小妮儿家人、旁白称呼，符合太行山一带对小姑娘的亲昵称呼感
成年妮儿 / 阿妮现实层女儿称“娘”，旁白可称“阿妮”
正式姓名待确认如果后续需要身份证式姓名，可另取，不影响主线

4. 12 集分集结构

集数标题核心事件开头钩子结尾追更点
01女儿要听娃娃仙姜生缠着要听娃娃仙，姜苗说自己都会背；回忆里姥爷救道士，道士赠五辫布娃娃后天亮消失，娃娃交到婴儿小妮儿手里红布打开，露出干裂的姜阿妮旁白：“它第一次保人，是从我五岁那年开始的。”
02祖奶带小妮儿进竹林下午小妮儿抱着五辫娃娃玩，祖奶从西屋门帘后探头观察；姥姥姥爷上山后，祖奶把她骗进竹林并推入深坑西屋门帘后，祖奶探头看小妮儿和娃娃祖奶回西屋装睡；姥姥姥爷天擦黑带糖回家开门，但本集不发现孩子不见
03深坑里的姐姐姥姥姥爷发现小妮儿不见后寻找；小妮儿在坑底哭到天黑，娃娃仙姐姐垂下长辫救她回家姥姥姥爷发现院里和屋里都没有小妮儿回家后娃娃少了一根辫子，姥姥姥爷决定分家
04烂肚子蛇小妮儿放羊被蛇咬、崴脚；看到蛇时脑海闪回以前听乡亲说“烂肚子”最爱躲草里毒蛇从草里扑出，脑海闪回“咬了人，半条命都要交出去”草丛动静越来越近，娃娃仙姐姐出现并用草蚂蚱转移她的恐惧
05羊自己回来了姐姐用槐树枝处理蛇伤，并把散开的羊群找回来；家里正式供起娃娃仙伤口和脚踝疼痛缓下来娃娃又少一根辫子，家里点香供奉
06看电影的人少女妮儿看露天电影回家，村西山坡听见陌生男人叫名，男人从草丛探出半张脸热闹电影散场后，妮儿独自走黑坡男人笑着说“南边有席，跟我走”
07拦路鬼进屋妮儿被拦路鬼纠缠，娃娃仙持扫把赶鬼被窝外有人绕床摸索上香时娃娃辫子再少，只剩两根
08曹老头的坟曹老头下葬后村里大旱，牲畜死亡，道士判断有旱魃圈里牲畜一夜死去棺材打开，红毛尸体露出
09火烧不化村民焚烧旱魃无效，妮儿夜里梦见山顶火光火堆烧了一下午，尸体不坏梦里娃娃仙被旱魃摔向火边
10辫子化剑娃娃仙揪下最后两根辫子化剑，跃入火堆斩旱魃娃娃仙满脸火光回头看她清晨娃娃破烂、辫子全无
11婚礼上的道士妮儿结婚回老家办酒，道士年轻模样归来，要取走娃娃酒宴后门口站着盘发男人道士拆开娃娃，取出一颗姜
12姜会发芽吗回到现实层，姜被种在姥姥姥爷墓前，女儿追问娃娃仙是否会回来清明墓前，泥土没有发芽妮儿说“这世上没有什么是不可能的”

11.5 生图提示词交接标准

生图提示词阶段按 storyboard-image-prompts 工作流执行，顺序固定为：

世界观约束 -> 场景美术规则 -> 角色设定锚点 -> 当前镜头动作/情绪 -> 平台化提示词
项目要求
视觉圣经每集提示词文件开头必须包含全片风格锚点、角色锚点、统一负面提示词
镜头覆盖分镜表每一行都必须有对应生图提示词
角色一致性复用 03_角色设定/角色固定提示词.md 的锚点，不临时改年龄、服装、脸型
世界观可视化“真实恐怖志怪”“娘子关一带旧村”等抽象词必须落成关隘山路、河沟、土墙/石墙旧屋、竹林、灶火、供桌、旧布娃娃等可见元素
平台选择中文短剧/漫剧优先即梦、可灵、通义万相、混元、FLUX/SD；正式付费批量前需按当时平台能力复核
风险标注涉及儿童、恐怖、墓地、真实人物肖像、上传参考图时必须标注审核和隐私风险

提示词表默认字段：

字段用途

画面目的说明这一镜为什么要出图
推荐平台/模型写明平台和简短理由
主提示词描述画面中可见主体、动作、环境、年代、画幅
角色锚点重复稳定人物描述
场景/镜头景别、角度、构图
光影色彩时间、光源、色调
负面提示词排除变形、文字水印、血腥过度、真实肖像等
参数/备注画幅、seed、参考图、LoRA、一致性提醒

11.8 小白照做版

  1. 把故事原文放进 09_素材与参考/。
  2. 让 AI 先整理成 <项目名>大纲.md。
  3. 确认集数、时长、画幅、画风、平台。
  4. 确认讲故事方式和主角称呼。
  5. 生成或更新 <项目名>_PRD.md。
  6. 同步生成 <项目名>_PRD.html。
  7. 更新 项目状态.md。
  8. 再开始做角色、剧本、分镜和提示词。
