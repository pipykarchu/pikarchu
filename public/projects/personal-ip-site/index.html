1. 需求诊断与概念对齐

视角结论
用户视角访客需要在很短时间内看懂“皮玺玉是谁、做过什么、能帮我什么、怎么联系”。
商业视角先用免费作品和 Skill 展示能力，再引导定制咨询、工作流外包、培训或私域沟通。
技术视角轻量 Web App：Vue 3 前台、content JSON 数据驱动、Node API 处理留言/埋点/客服/后台。
概念版： 一个数据驱动的个人 IP 作品集站点，用“皮玺玉 × AI 貔貅”的统一叙事展示 AI 工作流作品，并承接联系、咨询和商业服务转化。
目标用户招聘方、潜在客户、同行 / 产品经理 / AI 爱好者、未来学员。
核心场景30 秒内理解定位，浏览代表作品，通过客服或联系弹窗进一步咨询。
当前阶段面试演示作品集版，重点证明产品思考、AI 工作流实践和工程落地能力。
后续阶段小程序商业化版，单独立项设计交易、服务交付、会员或课程体系。
本版不做在线支付、用户登录、复杂 CMS、自动报价、多人客服坐席、多语言站点。
成功标志访客能在 1 分钟内获得个人介绍、作品概览或联系入口。

2. 背景与目标

皮玺玉已经沉淀了多个跨领域 AI 实践项目，包括求职作战中心、采购预警看板、PRD Skill、漫剧生产工作流、脚本 Skill、健康管理想法等。如果这些内容只以零散链接、截图或文档存在，访客难以快速理解整体能力，也不利于后续商业转化。

当前评审对象

当前主线：面试演示作品集版

本 PRD 当前只评审这一版。目标是展示个人 IP、AI 项目能力、产品思考、工程落地和复盘能力。

  • 主要受众：面试官、HR、合作方、同行、潜在客户。
  • 产品形态：Web 作品集站点 + 本地/线上 API + 演示后台。
  • 演示重点：首屏定位、作品分区、Live 项目、智能客服、留言后台、PRD 规划。
  • 验收重点：看懂能力、项目可信、演示顺畅、可讲清楚方法论。

后续方向：小程序商业化

只作为下一阶段规划，不进入当前验收。

  • 触发条件：面试反馈、私域咨询、真实需求验证。
  • 需要单独重做：账号、支付、订单、会员、交付、合规。

7. 需求详情

R1 首屏与导航首屏展示 greeting、slogan、intro；主 CTA 滚动到作品区，次 CTA 打开联系弹窗；主题切换需本地持久化。
R2 作品展示项目按 category 分区、priority 排序；状态只允许 live/wip/idea/paid；Live 项目可跳转 Demo。
R3 服务区由 services.json 驱动；免费资源滚动到作品区，付费服务打开联系弹窗；不展示支付按钮。
R4 联系弹窗留言表单至少填写 contact 或 message；成功后写入 contacts.jsonl；邮件提醒依赖 Resend 或 SMTP 配置。
R5 智能客服前端发送最近 8 条消息；后端读取站点资料；资料外问题说明不确定并引导留言；未配置 Key 时兜底。
R7 运营后台后台入口为 /admin；支持登录、查看留言、修改状态、查看事件和安全配置状态。
R11 内容维护 SOP改个人资料、作品、分区、服务、二维码和吉祥物时优先改 content 或 public 资产，不改业务代码。
R12 面试演示脚本按首页定位、作品区、Live 项目、智能客服、留言后台、PRD 规划的顺序演示。

10. 隐私与数据说明

当前版本会收集留言数据、访问行为数据和客服问题摘要。面试演示时可说明这是为了展示运营闭环；公开上线前需要补正式隐私说明。

数据类型收集内容存储位置用途保留策略
留言数据称呼、联系方式、留言内容、来源、时间data/contacts.jsonl跟进合作、招聘、咨询线索待确认；建议 180 天复盘清理
留言状态未读、已读、已回复、已归档data/contact_status.jsonl管理跟进进度跟随留言保留
访问行为事件类型、路径、来源、会话 ID、User-Agent、时间data/events.jsonl分析访问和转化待确认；建议 90 天复盘清理
客服问题摘要问题前 80 字data/events.jsonl分析访客关心什么避免记录敏感信息
AI 上下文最近 8 条对话消息请求时发送给 AI 服务生成回答当前不长期保存完整对话
最小收集面试演示版只收集留言、基础行为和问题摘要，不做用户画像和跨站追踪。
删除机制公开上线前应补“用户要求删除留言时如何处理”的流程。

12. 后续小程序商业化边界（非本期范围）

当前版本以“面试演示作品集”为主，商业转化只做轻量承接，不做交易闭环。后续小程序商业化是下一阶段方向，需要单独 PRD，不进入本期验收。

16. 风险与待确认

面试演示前最需要处理的是演示路径、核心项目素材和 AI 兜底；公开上线前最需要处理的是默认凭据、固定密钥、邮件提醒配置、隐私说明和部署域名。
编号风险/问题影响处理建议状态
Q1正式域名和部署平台未确认影响上线地址、SEO、分享链接确认部署平台待确认
Q2后台默认账号密码存在上线风险可能被他人登录后台查看留言配置 ADMIN_USER、ADMIN_PASS、ADMIN_SECRET待确认
Q3邮件提醒缺少 SMTP 或 Resend 配置站主可能不能及时收到留言提醒补邮件服务环境变量待确认
Q4二维码和简历 PDF 资产待补私域转化和招聘转化受影响上传真实二维码和简历待确认
Q7留言和埋点基于 JSONL 文件多实例部署可能不稳定访问量增长后迁移到数据库待确认
Q8隐私说明未明确收集联系方式和行为数据存在合规风险上线前补隐私声明待确认
Q10面试现场网络或 AI 服务不可用智能客服演示可能失败准备无 Key 兜底讲法和本地截图/录屏待确认
Q11作品集版与商业化小程序边界混淆面试官可能追问交易闭环和商业数据明确当前版本用于能力展示，商业化另行立项已确认
Q12AI 成本和 ROI 没有真实数据无法准确证明投入产出先保留追踪框架，后续从平台账单或会话统计补数据待确认
