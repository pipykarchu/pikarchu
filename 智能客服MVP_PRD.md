# 个人网站智能客服 MVP PRD

## 1. 背景

当前个人网站已经具备个人介绍、作品展示、联系方式和留言能力，但访客需要自己浏览页面寻找信息。新增智能客服后，访客可以用对话方式快速了解站点主人、代表作品和联系方式，降低理解成本，提升作品集演示效果。

## 2. 产品目标

| 项目 | 内容 |
| --- | --- |
| 产品名称 | 个人网站智能客服 |
| 使用场景 | 个人作品集网站访客咨询 |
| 目标用户 | 访问个人网站的招聘方、合作方、朋友或潜在客户 |
| 核心目标 | 让访客通过简单对话了解个人介绍、作品介绍和联系方式 |
| 成功标志 | 访客能在 1 分钟内通过聊天获得个人介绍、作品概览或联系入口 |

## 3. MVP 范围

### 本版本要做

| 功能 | 说明 | 优先级 |
| --- | --- | --- |
| 智能客服浮窗 | 页面右下角显示客服入口，点击后打开聊天窗口 | P0 |
| 资料问答 | 基于 `content/profile.json`、`content/projects.json`、`content/services.json` 回答问题 | P0 |
| 留言/联系入口 | 客服窗口内提供“留言 / 联系”按钮，复用现有联系弹窗 | P0 |
| 兜底回复 | 未配置 API Key 或 AI 不确定时，引导用户留言 | P0 |
| 访问频控 | 后端对聊天接口做简单限流，避免被刷接口 | P1 |

### 本版本不做

| 不做项 | 原因 |
| --- | --- |
| 人工客服坐席 | 当前只是作品集演示，不需要实时人工接待 |
| 用户登录 | 访客咨询不需要账号体系 |
| 长期聊天记录 | MVP 只需要当前会话体验 |
| 复杂知识库后台 | 当前内容已经由 `content/*.json` 驱动 |
| 自动报价 | 避免 AI 编造价格，合作细节引导留言 |

## 4. 用户流程

1. 访客打开个人网站。
2. 点击右下角客服入口。
3. 选择推荐问题，或手动输入问题。
4. 系统把问题发送到后端 `/api/chat`。
5. 后端读取个人资料和作品资料，调用 AI 模型生成回答。
6. 如果访客想继续联系，点击“留言 / 联系”。
7. 系统打开现有联系弹窗，访客可留下联系方式或查看二维码。

## 5. 核心需求

### 5.1 智能客服浮窗

| 字段 | 说明 |
| --- | --- |
| 入口位置 | 页面右下角 |
| 默认状态 | 收起，只显示吉祥物按钮 |
| 展开状态 | 显示聊天标题、消息区、快捷问题、输入框、联系按钮 |
| 移动端要求 | 不遮挡主要内容，宽度适配屏幕 |

验收标准：

- 点击吉祥物按钮能打开聊天窗口。
- 再次点击关闭按钮能收起聊天窗口。
- 输入长文本时不撑破布局。
- 手机和桌面端都能正常使用。

### 5.2 资料问答

| 字段 | 说明 |
| --- | --- |
| 数据来源 | `content/profile.json`、`content/projects.json`、`content/services.json` |
| 回答范围 | 个人介绍、作品介绍、服务内容、联系方式 |
| 限制规则 | AI 不允许编造未提供的信息 |
| 默认语言 | 简体中文 |

验收标准：

- 问“你是谁”时，能回答个人介绍。
- 问“有哪些作品”时，能概括作品列表。
- 问“怎么联系”时，能引导到留言/联系入口。
- 问资料外问题时，应说明不确定，并引导留言。

### 5.3 留言/联系入口

| 字段 | 说明 |
| --- | --- |
| 入口文案 | 留言 / 联系 |
| 行为 | 打开现有 `ContactModal` |
| 数据提交 | 复用现有 `/api/contact` |

验收标准：

- 点击客服窗口里的“留言 / 联系”能打开联系弹窗。
- 原有留言表单不受影响。

## 6. 技术方案

| 模块 | 方案 |
| --- | --- |
| 前端 | Vue 3 新增 `ChatWidget.vue` |
| 后端 | Node 原生 HTTP 新增 `/api/chat` |
| AI 接口 | 阿里云百炼 OpenAI 兼容接口 |
| 默认模型 | `qwen-turbo` |
| 配置文件 | `.env` |
| Key 安全 | API Key 只在后端读取，不进入前端代码 |

## 7. 环境变量

```env
DASHSCOPE_API_KEY=你的真实API_KEY
AI_MODEL=qwen-turbo
AI_API_URL=https://dashscope.aliyuncs.com/compatible-mode/v1/chat/completions
```

## 8. 风险与对策

| 风险 | 影响 | 对策 |
| --- | --- | --- |
| API Key 泄露 | 产生费用或被滥用 | 只放 `.env`，不要提交 GitHub |
| AI 编造信息 | 误导访客 | 系统提示限制只基于站点资料回答 |
| 模型无权限 | 聊天不可用 | 使用已验证可用的 `qwen-turbo` |
| 被频繁调用 | 成本上升 | 后端加聊天限流 |
| 内容乱码或资料不完整 | 回答质量下降 | 后续清理 `content/*.json` 文案 |
